# Edge 金十数据侧边栏插件 — 调试指南

## 一、当前框架的工作机制总结

### 1.1 侧边栏注入机制
- `content_script.js` 在所有页面（`<all_urls>`）自动执行，创建带 `z-index: 2147483646` 的 `position: fixed` 容器
- 容器内含 `<iframe>`，指向 `chrome.runtime.getURL('sidebar/sidebar.html')`
- 页面内容通过 `document.documentElement.style.marginRight` 向左推移，避免遮挡

### 1.2 通信机制（改造前）
```
sidebar.js → chrome.runtime.sendMessage(type: 'GET_MOCK_DATA') → background.js → sendResponse(模拟数据)
sidebar.js → window.parent.postMessage(type: 'TOGGLE') → content_script.js → chrome.runtime.sendMessage → background.js → 广播所有标签页
```

### 1.3 当前数据状态
- **全部使用模拟数据**：`background.js` 中的 `generateMockData()` 函数生成假数据
- **快讯**：8条静态假新闻，时间相对于当前时间偏移
- **指数**：7个假指数数据
- **商品**：6个假商品价格
- **日历**：5个假财经日历事件
- sidebar 每30秒通过 `chrome.runtime.sendMessage({type:'GET_MOCK_DATA'})` 拉取一次

### 1.4 通信机制（改造后）
```
sidebar.js (iframe) ──postMessage──→ content_script.js ──port.postMessage──→ background.js
                                                                    ↓ 轮询金十API
sidebar.js (iframe) ←──postMessage── content_script.js ←──port.postMessage── background.js (缓存数据)
```

## 三、数据接入步骤

### Step 1: 修改 background.js — 添加 Jin10FlashPoller 快讯轮询器

**目标**：在 Service Worker 中实现快讯轮询，去重后广播给所有页面。

**关键实现**：
- 每15秒 GET 请求 `https://flash-api.jin10.com/get_flash_list?channel=-8200&vip=1`
- 请求头：`x-app-id: bVBF4FyRTn5NJF5n`，`x-version: 1.0.0`
- 使用 `Set` 存储已见过的 `id`，过滤重复快讯
- 过滤广告：`item.extras?.ad === true` 的快讯丢弃
- 过滤空内容：`!item.data?.content` 的快讯丢弃
- `seenIds` 超过 2000 个时截断保留最近 1000 个
- 新快讯缓存到 `cachedFlash` 数组（最多 500 条）

**验证方法**：
1. 打开 `edge://extensions/` → 点击 Service Worker 的"检查视图"
2. 在 Console 中应看到：`[Jin10FlashPoller] 开始轮询快讯数据`
3. 等待 15 秒后应看到：`[Jin10FlashPoller] 获取到 N 条新快讯`
4. 确认没有 CORS 错误

### Step 2: 修改 background.js — 添加 Jin10CalendarService 财经日历服务

**目标**：获取当天的财经日历数据。

**关键实现**：
- 启动时立即 fetch `https://cdn-rili.jin10.com/web_data/{yyyy}/daily/{MM}/{dd}/economics.json`
- 按 `pub_time` 升序排序
- 缓存到 `cachedCalendar` 数组
- 不需要 API Key

**验证方法**：
1. 在 Service Worker Console 中应看到：`[Jin10CalendarService] 获取到 N 条日历事件`
2. 确认返回数据包含 `country`、`name`、`star`、`actual`、`consensus`、`previous`、`pub_time`、`affect` 字段

### Step 3: 修改 background.js — 添加 Port 连接处理和消息路由

**目标**：让 background.js 能接收 content_script 的数据请求并推送更新。

**关键实现**：
- `chrome.runtime.onConnect` 监听 `jin10-sidebar` 端口连接
- 将 port 存入 `Set<Port>`，断开时移除
- 端口消息路由：
  - `GET_FLASH` → 返回所有缓存快讯
  - `GET_CALENDAR` → 返回缓存日历数据
- FlashPoller 获取新数据后遍历所有 port 广播 `FLASH_UPDATE`
- 保留原有的 `onMessage` 处理器（sidbar 状态管理）

**验证方法**：
1. 在 Service Worker Console 中确认 port 连接日志
2. 在任意页面打开 F12 Console（内容脚本侧），确认收到 `FLASH_UPDATE` 消息

### Step 4: 修改 content_script.js — 建立 Port 连接并转发消息

**目标**：content_script 作为 sidebar iframe 和 background 之间的中继。

**关键实现**：
- 页面加载时调用 `chrome.runtime.connect({name: 'jin10-sidebar'})` 建立长连接
- 监听 port 的 `onMessage`，收到 `FLASH_UPDATE` 后通过 `iframe.contentWindow.postMessage` 转发
- 监听 sidebar iframe 的 `postMessage`，将以 `DATA_` 或 `GET_` 开头的请求类型转发到 background
- 使用 `requestId` 关联请求和响应
- 保留原有的 TOGGLE/RESIZE postMessage 处理和 state 同步

**验证方法**：
1. 打开任意网页按 F12 → Console 标签页
2. 应看到：`[ContentScript] 已连接到 Background Service Worker`
3. 应看到：`[ContentScript] 转发 FLASH_UPDATE 到 sidebar，N 条新快讯`
4. 确认 sidebar 收到数据并正常渲染

### Step 5: 修改 sidebar/sidebar.js — 渲染真实数据

**目标**：用真实金十数据替换模拟数据渲染。

**关键实现要点**：

**快讯渲染：**
- `item.time` 格式为 `"2026-04-29 15:49:39"`，提取时间部分（后8位 `HH:MM:SS`）
- `item.data.content` 包含 HTML 标签（如 `<b>`、`<font>`），用正则 `/<[^>]*>/g` 清理
- `item.important >= 1` → 添加 `flash-important` 类（左侧红色边框，文字红色加粗）
- 显示加载状态：`<div class="loading">正在加载快讯数据...</div>`
- 无数据时：`<div class="empty-state">暂无快讯数据</div>`

**日历渲染：**
- `star` 字段是字符串（`"1"`-`"5"`），转为数字后用 ★☆ 显示星级
- `pub_time` 格式 `"2026-04-29 20:30:00"`，提取时间部分
- 显示：时间、国家、事件名称、前值/预测/公布值、星级
- `affect` 字段可能是中文描述（如"利多金银"），显示在事件右侧

**指数/商品渲染：**
- 由于 MCP API 需要 Bearer Token，先检查 `chrome.storage.local` 中是否有 `jin10_api_token`
- 如果没有 Token → 显示提示"请配置 API Token 以获取实时行情"，附带链接说明
- 如果已有 Token → 通过 MCP JSON-RPC 获取行情数据（后续步骤实现）

**通信改为 postMessage：**
- 不再使用 `chrome.runtime.sendMessage` 拉取数据
- 改为 `window.parent.postMessage({type: 'GET_FLASH', requestId})` 请求数据
- 监听 `window.onmessage` 接收来自 content_script 的 `FLASH_UPDATE` 和 `DATA_*` 响应

**验证方法：**
1. 打开任意网页，侧边栏应显示出来
2. "快讯"标签应显示真实的金十财经新闻
3. 重要快讯应有红色左边框和红色文字
4. "日历"标签应显示当天的财经日历事件
5. 时间应提取正确，HTML标签应已清理
6. "指数"标签如果没有token应显示配置提示
7. 底部"数据更新时间"应显示最新刷新时间

## 五、常见错误与解决方案

### 错误 1：`Access to fetch ... has been blocked by CORS policy`
- **原因**：在 content_script 或 sidebar.js 中直接 fetch 金十 API
- **解决**：确保所有 fetch 请求在 background.js (Service Worker) 中执行
- **检查**：搜索所有 `.js` 文件，确认没有 fetch 调用出现在 background.js 之外

### 错误 2：`fetch failed` / `TypeError: Failed to fetch`
- **原因**：网络连接问题，或 API 地址变更
- **解决**：
  1. 在 Service Worker Console 中手动测试 fetch
  2. 检查 `manifest.json` 的 `host_permissions` 是否包含 API 域名
  3. 尝试在浏览器中直接访问 API URL 确认可用性

### 错误 3：快讯列表为空
- **原因**：可能过滤条件过于严格，或 API 返回格式变化
- **解决**：
  1. 在 Service Worker Console 中打印 `json.data` 查看原始数据
  2. 暂时注释掉广告过滤和空内容过滤，观察原始数据
  3. 检查 `seenIds` Set 是否与快讯 ID 格式匹配

### 错误 4：`port.postMessage is not a function`
- **原因**：port 连接已断开或变量作用域错误
- **解决**：
  1. 检查 port 连接是否在 content_script 的顶层作用域
  2. 确认 `chrome.runtime.connect()` 返回值被正确保存
  3. 检查 `port.onDisconnect` 事件，确认断连原因

### 错误 5：`Cannot read properties of null`
- **原因**：DOM 元素未加载完成就执行操作
- **解决**：
  1. 确保 `injectSidebar()` 在 DOM 就绪后调用
  2. 在 sidebar.js 中使用 null 检查：`if (element) { ... }`

### 错误 6：消息格式不匹配
- **原因**：sidebar 期望的字段名与实际发送的不一致
- **解决**：
  1. 在 content_script Console 中打印所有转发消息：`console.log('转发到sidebar:', msg)`
  2. 在 sidebar Console 中打印所有接收消息：`console.log('收到消息:', event.data)`
  3. 对比数据结构，统一字段命名

### 错误 7：sidebar Console 无日志输出
- **原因**：iframe 的 Console 需要在侧边栏上右键打开独立检查窗口
- **解决**：
  1. 在侧边栏内部右键 → 检查（如果可用）
  2. 或在 F12 的 Sources 面板中找到 `sidebar.html`，设置断点调试
  3. 或在 content_script 中添加转发日志：打印所有向 iframe 发送的消息
